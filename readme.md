Аналіз складності

Жадібний алгоритм :

-Часова складність: O(amount / min_coin)

-Дуже швидкий на практиці

-Не гарантує оптимальність для довільних наборів монет

Динамічне програмування :

-Часова складність: O(amount × n)

-Завжди знаходить мінімальну кількість монет

-Повільніший і потребує більше памʼяті

Висновок :

Жадібний алгоритм є найкращим вибором для касових систем зі стандартними номіналами монет, оскільки він працює швидко та ефективно навіть для великих сум.
Алгоритм динамічного програмування є універсальним рішенням, що гарантує мінімальну кількість монет, однак його використання для великих сум є менш доцільним через вищу часову та просторову складність.
